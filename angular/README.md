# mcaap-onboarding-angular

Angular **standalone-component** wrapper for the `<mcaap-onboarding>` web component,
generated by [`@stencil/angular-output-target`](https://stenciljs.com/docs/angular).

The proxy components (`projects/mcaap-onboarding/src/lib/stencil-generated/`) are
produced by the **Stencil build in the parent package** — they are not hand-written and
are git-ignored.

## Build

```bash
# 1. from the parent package (../), build Stencil — this emits dist/ AND the proxies here
cd .. && npm install && npm run build

# 2. build this Angular library with ng-packagr
cd angular && npm install && npm run build   # -> dist/mcaap-onboarding-angular
```

The library declares `@angular/core` and `mcaap-onboarding` as **peer dependencies** —
the consuming app provides both.

## Use it (standalone)

Because the proxy self-registers the custom element (`includeImportCustomElements`),
you only import the component — no `defineCustomElements()` call:

```ts
import { Component } from '@angular/core';
import { McaapOnboarding } from 'mcaap-onboarding-angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [McaapOnboarding],
  template: `
    <button (click)="onb.open()">Help</button>
    <mcaap-onboarding
      #onb
      [debug]="false"
      [navigateHandler]="go"
      (mcaapTourCompleted)="track($event)"
      (mcaapNavigate)="route($event)">
    </mcaap-onboarding>
  `,
})
export class AppComponent {
  go = (page: string) => this.router.navigate([page]);
  track = (e: CustomEvent<{ id: string; title: string }>) => analytics.track('tour_done', e.detail);
  route = (e: CustomEvent<{ page: string }>) => {};
  constructor(private router: Router) {}
}
```

Add `CUSTOM_ELEMENTS_SCHEMA` is **not** required — the wrapper is a real Angular
component. Inputs: `debug`, `navigateHandler`. Methods: `open()`, `start(id)`,
`close()`, `validate()`. Outputs: `mcaapTourStarted`, `mcaapTourStep`,
`mcaapTourCompleted`, `mcaapTourSkipped`, `mcaapNavigate`.
